Entrada binaria doble

Cada dato DNP3 dentro de la base de datos de una estación remota se denomina Punto y se identifica por un Índice. El valor actual del punto se denomina valor estático. El punto puede generar eventos por cambios de valor o ante la recepción de comandos. Los eventos pueden almacenarse en una de las 3 clases que permite DNP3 (Clase 1, Clase 2 o Clase 3).

Clases en DNP3

Class 0
Static Value

Class 1
Events

Class 2
Events

Class 3
Events

Las clases en DNP3 permiten agrupar los eventos en tres niveles diferentes (recurso útil para priorizar el esquema de peticiones del maestro). Las peticiones por clase permiten el polling por eventos. Un caso excepcional es la clase 0, mediante la cual el maestro puede solicitar al esclavo el envío de todos los valores estáticos configurados en su base de datos.

Para identificar el formato de un punto de la base de datos de DNP3 dentro de un mensaje se usan 3 propiedades:

Grupo: Identifica el tipo de información que viaja en el mensaje. Por ejemplo, valor actual de entrada binaria.

Variación: formato de la información que indica si viaja la calidad y/o marca de tiempo o el formato de los datos analógos

Índice: dirección en la base de datos.

Interoperabilidad

Nivel 1: Esta es la funcionalidad mínima que debe implementar un maestro o una estación remota para que sea compatible con DNP3. La funcionalidad será bastante básica y solo está destinada a un pequeño dispositivo de campo.

Nivel 2: Esto agrega varios códigos de función y objetos para admitir una RTU simple o un concentrador de datos.

Nivel 3: Esto agrega códigos de función y objetos adicionales, especialmente habilitar / deshabilitar los mensajes no solicitados, para admitir RTU / data concentrator complejos.

Nivel 4: Esto agrega características de uso poco frecuente, como variaciones analógicas de punto flotante, entradas binarias de doble bit y atributos de dispositivo.

Capa de Aplicación

La capa de aplicación define Funciones (servicios) que se intercambian entre estaciones controladoras (Master) y estaciones remotas (Outstation),en esta capa tambien se realiza la fragmentacion a nivel de usuario.
Las funciones básicas de DNP3 son:

ASSIGN
